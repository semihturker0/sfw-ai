## English

### Project Overview
sfw-ai is a research-oriented framework developed in C++ designed to investigate Windows kernel internals and advanced system instrumentation. The core philosophy of the project is to decouple the bypass logic from the execution environment. This is achieved through a "Static Core / Dynamic Payload" architecture, where a stable kernel driver acts as a host for polymorphic code fragments generated in real-time by an external intelligence module.

### Legal and Ethical Disclaimer
This software is developed strictly for educational purposes and authorized security auditing. Operating within Ring-0 (Kernel Mode) carries significant risks, including system instability and permanent data loss. The author assumes no liability for damages resulting from the use or misuse of this framework. Users are responsible for complying with local laws and regulations.

### Technical Architecture
The framework is divided into three primary components:

1. **Kernel Interface (Static Core):** A C++20 driver responsible for memory management, IOCTL handling, and providing a secure execution context for incoming payloads.
2. **Intelligence Orchestration Layer:** A user-mode module that analyzes system telemetry and generates context-aware instruction streams.
3. **Execution Engine:** A specialized component within the driver that maps and executes dynamically generated bytecode while maintaining system stability and stealth.

### Specifications
* **Implementation:** C++20 / x64 Assembly.
* **SDK Requirements:** Windows Driver Kit (WDK) 10/11.
* **Communication:** Optimized IOCTL interface with encrypted buffers.
* **Focus Areas:** Direct Kernel Object Manipulation (DKOM), instruction stream polymorphism, and memory protection analysis.

## Türkçe

### Proje Özeti
sfw-ai, Windows çekirdek (kernel) iç yapılarını ve gelişmiş sistem araçlarını incelemek amacıyla C++ ile geliştirilmiş bir araştırma çerçevesidir. Projenin temel felsefesi, bypass mantığını yürütme ortamından ayrıştırmaktır. Bu, "Statik Çekirdek / Dinamik Yük" mimarisi ile sağlanır; burada sabit bir kernel sürücüsü, harici bir zeka modülü tarafından gerçek zamanlı olarak üretilen polimorfik kod parçacıkları için bir taşıyıcı görevi görür.

### Yasal ve Etik Uyarı
Bu yazılım tamamen eğitim ve yetkili güvenlik denetimleri amacıyla geliştirilmiştir. Ring-0 (Kernel Mode) seviyesinde çalışmak, sistem kararsızlığı ve kalıcı veri kaybı dahil olmak üzere önemli riskler taşır. Yazar, bu çerçeve yazılımın kullanımından veya kötüye kullanımından doğacak zararlar için hiçbir sorumluluk kabul etmez. Yerel yasalara ve düzenlemelere uyum sağlamak kullanıcının sorumluluğundadır.

### Teknik Mimari
Framework üç ana bileşene ayrılmıştır:

1. **Çekirdek Arayüzü (Statik Çekirdek):** Bellek yönetimi, IOCTL işleme ve gelen yükler için güvenli bir yürütme bağlamı sağlamaktan sorumlu C++20 sürücüsü.
2. **Orkestrasyon Katmanı:** Sistem telemetrisini analiz eden ve duruma özel komut akışları üreten kullanıcı modu (user-mode) modülü.
3. **Yürütme Motoru:** Sürücü içerisinde yer alan, dinamik olarak üretilen bytecode'u sistem stabilitesini ve gizliliği koruyarak haritalayan ve çalıştıran özel bileşen.

### Teknik Detaylar
* **Uygulama:** C++20 / x64 Assembly.
* **SDK Gereksinimleri:** Windows Driver Kit (WDK) 10/11.
* **İletişim:** Şifrelenmiş tampon belleklere sahip optimize edilmiş IOCTL arayüzü.
* **Odak Alanları:** Doğrudan Çekirdek Nesne Manipülasyonu (DKOM), komut akışı polimorfizmi ve bellek koruma analizi.
